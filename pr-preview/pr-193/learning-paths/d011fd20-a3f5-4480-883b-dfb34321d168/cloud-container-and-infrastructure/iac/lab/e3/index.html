Create a Google Cloud Compute Engine Instance Using Terraform

3 minutes read
  1. Create a directory to store our Terraform code:
mkdir -p ~/code
  1. Create a Terraform configuration file and save the file as main.tf with the following configuration:
provider "google" {
  project = "<ADD YOUR PROJECT ID HERE>"
  region  = "us-central1"
  zone    = "us-central1-a"
}

resource "google_compute_instance" "my_instance" {
  name         = "my-instance"
  machine_type = "n1-standard-1"
  zone         = "us-central1-a"

  boot_disk {
    initialize_params {
      image = "debian-cloud/debian-10"
    }
  }

  network_interface {
    network = "default"
    access_config {
      // Ephemeral IP
    }
  }
}

Note: Replace " ADD YOUR PROJECT ID HERE " with your Google Cloud project ID. You can also customize other parameters like the instance name, machine type, zone, and image.

  1. Initialize Terraform, navigate to the directory containing your Terraform configuration file (main.tf). Initialize the directory which will download the necessary providers.
cd ~/code
terraform init

You can further extend your configuration to include additional resources or customize the instance properties as needed.
